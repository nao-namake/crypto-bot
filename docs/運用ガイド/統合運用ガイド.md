# 第5部: 戦略プリセット差し替え（Phase 57）

## プリセットシステム概要

戦略設定はプリセット形式で管理されています：

```
config/core/strategies/
├── active.yaml              ← 現在アクティブなプリセットを指定
└── presets/
    ├── phase56.yaml         ← Phase 56設定（本番運用中）
    ├── phase57_4.yaml       ← Phase 57.4設定
    └── optuna_YYYYMMDD.yaml ← Optuna最適化結果（自動生成）
```

## Optuna結果の保存先

Optuna最適化結果は2箇所に保存されます：

```
config/
├── optimization/
│   └── results/
│       └── latest_optimization.json     ← 生データ（詳細パラメータ）
└── core/
    └── strategies/
        └── presets/
            └── optuna_YYYYMMDD.yaml      ← 運用プリセット（自動変換）
```

## 戦略差し替え手順（3ステップ）

### Step 1: 新プリセットの確認・準備

```bash
# 利用可能なプリセット一覧
ls config/core/strategies/presets/

# プリセット内容確認
cat config/core/strategies/presets/<preset_name>.yaml

# パフォーマンス情報確認（各プリセットに記載）
# - win_rate: 勝率
# - profit_factor: プロフィットファクター
# - total_pnl: 総損益
```

### Step 2: アクティブプリセットの変更

```bash
# active.yaml を編集
# active_preset: <新しいプリセット名> に変更

# 例: Optuna最適化結果を適用
# active_preset: optuna_20251202
```

### Step 3: システム再起動・検証

```bash
# ペーパーテストで動作確認（5-10分）
bash scripts/management/run_safe.sh local paper

# 確認項目:
# ✅ 特徴量生成成功
# ✅ 戦略シグナル発火
# ✅ ML予測動作
# ✅ エラーログなし

# 停止
bash scripts/management/run_safe.sh stop

# 問題なければ本番デプロイ
git add config/core/strategies/active.yaml
git commit -m "chore: プリセット切り替え → <preset_name>"
git push origin main
```

## Optuna最適化からの差し替えフロー

```
1. GitHub ActionsでOptuna最適化実行
   gh workflow run "Optuna Parameter Optimization"
       ↓
2. 最適化完了（約3-4時間）
   → config/optimization/results/latest_optimization.json 保存
   → config/core/strategies/presets/optuna_YYYYMMDD.yaml 自動生成
       ↓
3. active.yaml の active_preset を変更
   active_preset: optuna_YYYYMMDD
       ↓
4. ペーパーテスト → 本番デプロイ
```

問題発生時は `active_preset` を前のプリセットに戻すだけ：

```yaml
# config/core/strategies/active.yaml

```

