## 固定金額TP設定（Phase 63.2: 累積手数料バグ修正済み）

`config/core/thresholds.yaml`の`position_management.take_profit.fixed_amount`セクションを編集

```yaml
position_management:
  take_profit:
    fixed_amount:
      enabled: true                     # 有効/無効切替
      target_net_profit: 500            # 目標純利益（円）
      include_entry_fee: true           # エントリー手数料を考慮
      include_exit_fee_rebate: true     # 決済手数料を考慮
      include_interest: true            # 利息を考慮
      fallback_entry_fee_rate: 0.0     # Phase 63.2: 常にこのレート使用（Maker 0%）
      fallback_exit_fee_rate: 0.0      # Phase 63.2: 常にこのレート使用（Maker 0%）
```

**計算例（Maker 0%）**:
```
目標純利益: 500円
エントリー手数料: 0円（fallback_entry_fee_rate=0.0で計算）
利息: 0円（Phase 63.2: 新規エントリー時点では常に0円）
決済手数料: 0円（fallback_exit_fee_rate=0.0で計算）

必要含み益 = 500 + 0 + 0 + 0 = 500円
TP価格 = エントリー価格 ± (500 / 数量)
```

**Phase 63.2修正**: bitbank APIの`unrealized_fee_amount`は集約ポジション全体の累積手数料であり、個別エントリーの手数料ではない。この値をTP計算に使用すると必要含み益が膨張する（500円→平均741円）ため、常にfallbackレートで個別計算する方式に変更。fee_dataはログ出力（参考値）のみ。

**注意事項**:
- `enabled: false`で%ベースTPに戻る（後方互換性維持）
- SL設定は%ベースのまま維持
- `fallback_entry_fee_rate`/`fallback_exit_fee_rate`の値がTP計算に直接使用される

python3 scripts/live/standard_analysis.py                 # 基本（24時間）
python3 scripts/live/standard_analysis.py --hours 48      # 期間指定

python3 scripts/live/standard_analysis.py --exit-code     # CI/CD連携（終了コード返却）

# 直接実行（CSV収集・レポート生成なし）
