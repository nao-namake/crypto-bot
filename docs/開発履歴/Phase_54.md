### 7.5 5分間隔実行 vs 15分足エントリーの整合性問題

**ユーザー指摘の重要な問題点**:

#### **現状の不整合**:
```

- 15分足の最初の5分間・10分間は不完全なデータで判定
```

```
15分足: [00:00-00:15] のローソク足

```

#### **対応オプション検討**:

**Option 1: 15分間隔実行に変更（推奨度: 中）**

- **影響**: バックテスト実行時に「CSVの列が不正」エラー発生
- **根本原因**: 手動データ準備時の操作ミス

#### **解決策**: run_backtest.sh機能拡張

**追加機能**:
1. **`--days <日数>` オプション**: 指定期間のデータを自動収集
2. **`--collect-only` オプション**: データ収集のみ実行（バックテスト省略）

```bash
# 引数解析
while [[ $# -gt 0 ]]; do
  case $1 in
    --days)
      DAYS="$2"
      shift 2
      ;;
    --collect-only)
      COLLECT_ONLY=true
      shift
      ;;
    *)
      PREFIX="$1"
      shift
      ;;
  esac
done

# データ収集（--days指定時）
if [[ -n "$DAYS" ]]; then
  echo "📊 データ収集開始: ${DAYS}日間"
  if python3 src/backtest/scripts/collect_historical_csv.py --days "$DAYS"; then
    echo "✅ データ収集完了"
  else
    echo "❌ データ収集失敗"
    exit 1
  fi
fi
```
